## Topic 1: Relational Database Design and Structure

**Coach:** Could you please paste a snippet of your SQL code that shows the tables you created for your organizations and service projects? Then tell me about the relationship between these tables.

**Isaac:** 
```sql
-- CREATE ORGANIZATION TABLE
CREATE TABLE organization (
    organization_id SERIAL PRIMARY KEY,
    name VARCHAR(150) NOT NULL,
    description TEXT NOT NULL,
    contact_email VARCHAR(255) NOT NULL,
    logo_filename VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- CREATE SERVICE PROJECT TABLE
CREATE TABLE service_project (
    project_id SERIAL PRIMARY KEY,
    organization_id INT NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT NOT NULL,
    location VARCHAR(255) NOT NULL,
    project_date DATE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (organization_id) REFERENCES organization(organization_id) ON DELETE CASCADE
);
```
The tables have a one-to-many relationship. One organization can have many service projects. The `service_project` table holds the foreign key `organization_id` referencing the `organization` table.

## Topic 2: Server-Side Data Access Logic (Node.js)

**Coach:** Could you please paste a snippet of your Node.js code that shows a function used to fetch data from one of these tables?

**Isaac:**
```javascript
import db from './db.js';

const getProjectsReport = async () => {
    const sourceTable = (await tableExists('service_project')) ? 'service_project' : 'projects';
    if (!(await tableExists(sourceTable))) {
        return { totalProjects: 0, projects: [], projectsByOrg: [], upcomingProjects: [] };
    }

    const hasLocation = await columnExists(sourceTable, 'location');
    const locationSelect = hasLocation ? 'p.location' : 'NULL AS location';

    const allProjectsQuery = `
        SELECT o.name AS organization_name, p.title, p.description, ${locationSelect}, p.project_date
        FROM public.${sourceTable} p
        JOIN public.organization o ON o.organization_id = p.organization_id
        ORDER BY o.name, p.project_date;
    `;

    const [totalRes, allRes, countRes, upcomingRes] = await Promise.all([
        db.query(totalCountQuery),
        db.query(allProjectsQuery),
        db.query(countByOrgQuery),
        db.query(upcomingQuery)
    ]);

    return {
        totalProjects: Number(totalRes.rows[0]?.total_projects ?? 0),
        projects: allRes.rows,
        projectsByOrg: countRes.rows,
        upcomingProjects: upcomingRes.rows
    };
};
```

**Key Discussion Points:**
- **Defensive Programming:** Using `tableExists` and `columnExists` to ensure the database schema matches expectations before querying.
- **Graceful Fallbacks:** Returning empty results instead of crashing if tables are missing.
- **Dynamic SQL:** Building queries dynamically (e.g., `${locationSelect}`) to handle potential schema variations.

**Coach:** Can you walk me through what happens step-by-step when a user visits the `/projects` URL?

**Isaac:**
1. **Request:** Express receives a GET request for `/projects`.

**Coach:** How are you displaying this data in your EJS template?

**Isaac:**
```ejs
<% report.projects.forEach(project => { %>
    <div class="project-card">
        <h3><%= project.title %></h3>
        <% const formattedDate = project.project_date ? new Date(project.project_date).toLocaleDateString('en-US', { year: 'numeric', month: 'short', day: 'numeric' }) : 'Date TBD'; %>
        <p><strong><%= formattedDate %>:</strong> <%= project.title %> (<%= project.organization_name %>)</p>
    </div>
<% }); %>
```

**Key Discussion Points:**
- **Date Formatting:** Formatting the date in the EJS template using `.toLocaleDateString()` to keep presentation logic separate from data logic.
- **Separation of Concerns:** The view is responsible for how data looks (presentation), while the model/controller handle data retrieval.
- **Graceful Fallbacks in UI:** Using a ternary operator to show "Date TBD" if a date is missing.
